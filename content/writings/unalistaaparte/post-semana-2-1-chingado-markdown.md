#

Creo que voy a cambiar el formato de título de los posts. De "semana X" a solo usar números romanos(?). Resulta más fácil la secuencia para justificar cualquier deseo de escribir, o mejor aun uso un [incrementing sequence numbering](https://en.wikipedia.org/wiki/Software_versioning#Incrementing_sequences), todos estos problemas ya fueron resueltos antes.

Pues el propósito de esta breve entrada, es probar mi nuevo editor de [Markdown](https://en.wikipedia.org/wiki/Markdown). ¿Por qué no solo mejor usar el editor de texto de Blogger/Medium o lo que sea? Pues por el temor a perder el contenido.

Razones listadas:

1. No hay nada más simple donde almacenar contenido, que en un archivo "txt", repito: NA-DA. 2. Cualquier editor, programa o navegador puede abrir un archivo de texto. Así que no hay problemas de compatibilidad, formatos extraños o que se yo. 3. La mejor manera de dar formato a un archivo de texto plano es usando Markdown.

Este formato me permite portabilidad del contenido que escriba, con el puedo generar codigo HTML usando herramientas a las que les huyo por el momento. En fin, es buena solución para un individuo que se preocupa por estas cosas.

Lo que si es un dolor de cabeza, son los editores de Markdown. Chingado Markdown, tan sencillo de usar, pero la funcionalidad que uno busca es casi imposible de encontrar. Descargue, instale y use CINCO aplicaciones, fucking cinco aplicaciones, para encontrar la indicada.

¿Qué termine usando al final? Un simple editor, bastante eficiente e interesante llamado: [Markdown Monster](https://markdownmonster.west-wind.com/).
